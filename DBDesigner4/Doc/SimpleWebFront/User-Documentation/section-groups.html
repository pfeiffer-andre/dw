Groups

On this tab you can create groups. Groups are containers for views and moreover only views assigned to a group are taken into account when the webpages are created.

Figure 2-5. Groups Tab

On the left hand side there is the Group management section. Very similiar to the view management section you can create, edit, and delete groups here.

  • Clicking the Create Group-Button it the normal way to create a new group. After clicking the button the Group-Editor-Dialog is showed.

  • Clicking the Delete Group-Button will delete the currently selected group in the list.

  • By clicking the Edit Group-Button you have the possibility to edit an already created group. Unsurprisingly the currently selected group in the listbox will be edited.

  • The Add Regions-Button provides a way to conveniently add all regions of the model as groups. This means that for each region in the model a group is created.

    Only the region names are retrieved from the model. What tables are in a region is not taken into account.

On the right hand side there is the Assignment View<->Group section. Here views are assigned to a group. As already mentioned before, a view has to be part of a group if the view should be accessible via the webpage.

If your model is rather small and you have no real use for groups, then just create a group named "Standard" or something and put all your views there.
