Introduction

I hope that SWF's user-interface is easy to use and understand. If you are not satiesfied, or have any comments on it, I would welcome your feedback and possible suggestions.

Figure 2-1. User Interface - Overview

The user interface is divided into five different tabs. Each tab lets you enter some specific information about the website you want to generate. To change between the tabs, use the navigation menu on the left side.

Regardless of the active tab, you can perform one of the three standard operations (Save, Generate Webpages, Close) by clicking on the respective button at the bottom of the user-interface or by choosing the correct menu entry from File.

  1. Save saves the current configuration into the model file. Currently there is no (visual) feedback if saving was successful, but the program will notify you in the event of a failure or problem.

    You also have to save the document in DBDesigner4 again or your changes will be lost.

  2. Close simply closes the application and returns to DBDesigner4.

  3. Create Webpages finally starts the process of creating webpages. If successful you will be notified.

    The Create Webpages button and the corresponding menu-entry is deactivated as long as necessary information for creating the webpages is still missing.

Here's a complete list of what fields, options need to get filled out, so that the Create Webpages Button and the corresponding menu entry is activated:

  • dbusername, dbpassword, dbhostname, dbname

  • webtitle, outputdirectory

  • One group with at least one view in it

Without this information no reasonable/working webpages could be created.
